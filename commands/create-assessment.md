Create an assessment for: $ARGUMENTS

## Steps

1. **Clarify assessment type** — formative (ongoing, low-stakes, guides instruction) or summative (end-of-unit, high-stakes, measures achievement). Ask if not clear.
2. **Search achievement standards** — use EasedUP MCP to retrieve achievement standards for the relevant year level and subject; these anchor the rubric
3. **Design the assessment task** — authentic where possible, aligned to learning intentions
4. **Generate the resource** — write the complete assessment as TipTap-compatible HTML

## Required Sections

- **Assessment Title**
- **Subject Area**
- **Year Level**
- **Curriculum Content Descriptions** — from EasedUP MCP, with codes
- **Achievement Standards** — from EasedUP MCP; used to set rubric performance levels
- **Assessment Type** — Formative or Summative (with brief rationale)
- **Purpose** — what knowledge or skills this assessment measures
- **Assessment Task** — description of what students will do; authentic contexts preferred
- **Student Instructions** — clear, accessible instructions for students
- **Assessment Criteria** — specific criteria used to evaluate performance
- **Rubric / Benchmark** — performance levels (e.g., Exceeding / Meeting / Approaching / Beginning, or A–E) with descriptors for each criterion
- **Feedback Mechanism** — how and when feedback will be provided (written, verbal, peer, rubric)
- **Differentiation** — adjustments for diverse learners while maintaining rigour

Output the complete assessment as HTML.
